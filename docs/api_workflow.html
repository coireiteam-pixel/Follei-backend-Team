🔐 Domain 1 - Identity & Auth

POST /auth/register
Create a new tenant with an admin user and return authentication tokens.
Request JSON
{
  "domain": "acme.com",
  "name": "Acme Corporation",
  "admin_email": "admin@acme.com",
  "admin_password": "SecurePass123!",
  "admin_first_name": "John",
  "admin_last_name": "Doe"
}
Response JSON
{
  "access_token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...",
  "token_type": "bearer"
}
🔄 Workflow:
  1. Client sends registration payload with tenant details
  2. System checks if domain already exists (409 if duplicate)
  3. System checks if email already exists (409 if duplicate)
  4. Creates tenant record
  5. Creates admin user with hashed password
  6. Returns JWT access token for immediate authentication
POST /auth/login
Authenticate user and return JWT token.
Request JSON
{
  "email": "admin@acme.com",
  "password": "SecurePass123!"
}
Response JSON
{
  "access_token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...",
  "token_type": "bearer"
}
🔄 Workflow:
  1. Client sends credentials
  2. System verifies email exists and password matches hash
  3. Checks if user is active (403 if inactive)
  4. Updates last_login_at timestamp
  5. Returns JWT token
GET /auth/me
Retrieve authenticated user's profile.
Headers
Authorization: Bearer
Response JSON
{
  "id": "11111111-1111-4111-8111-111111111111",

}

🔄 Common Workflow Patterns

1. Lead to Customer Conversion Flow

// Step 1: Create Lead
POST /api/leads
{
  "email": "lead@example.com",
  "full_name": "John Doe",
  "company": "Acme Corp",
  "status": "new"
}

// Step 2: Qualify Lead
POST /api/leads/{lead_id}/qualifications
{
  "framework_id": "...",
  "answers": {...}
}

// Step 3: Create Opportunity
POST /api/opportunities
{
  "lead_id": "{lead_id}",
  "name": "Acme Deal",
  "stage": "proposal"
}

// Step 4: Create Customer (after deal won)
POST /api/customers

🔑 Authentication

All protected endpoints require Bearer token in the Authorization header:
Authorization: Bearer eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...
🔄 Token Sources:
  1. POST /auth/login - Login with credentials
  2. POST /auth/register - Register new tenant
  3. POST /api/v1/auth/login - Alternative login
  4. POST /api/v1/auth/register - Alternative register
  5. POST /api/v1/auth/refresh - Refresh access token
